Paxos算法

  • 多个监督者、投票
  • 两个阶段：准备+接收
  • 两个原则：安全原则+存活原则
  • 多节点并发访问数据库，保持一致性：主从同步、异步、半同步复制（法定集合）
  • Paxos原理：法定集合性质：任意两个法定集合，必定存在一个公共的成员。该性质是Paxos有效的基本保障。
  • 二段提交原则（2PC）：prepare + accept
  • 提案排序办法：ID越大，优先级越高
  • 约束条件：acceptor接受收到的第一个提案；法定集合未审批过编号小于n的提案。
  • prepare阶段目的：核实value；根据ID阻塞早期提案。
  • 算法流程：
    • 提议者：询问 -> 预提议 -> 提议
    • 接收者：处理询问 -> 处理预提议 -> 处理提议
  • 诺言：诺言中包含了acceptor记录的v值。
  • 编号的重要性：
    • proposer忙于选择更大的编号提交proposal；
    • acceptor比较提交的proposal编号是否最大，只要编号确定了，所对应的value也就确定了。
  • 编号的活锁问题解决方案
    • 选举一个proposer作Leader（单点问题）
    • 随机、超时（可靠）
    • 递归使用Paxos
  • 算法容错性（持久化存储）：
    • proposer存储:已提交的最大proposal编号、决议编号（instance id）。
    • acceptor存储:已承诺（promise）的最大编号、已接受（accept）的最大编号和value、决议编号。
    • learner存储:已学习过的决议、决议编号。
